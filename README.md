# frontend

## Commit Header Convention

| Type       | Description                      |
|------------|----------------------------------|
| `feat`     | 새로운 기능 추가                  |
| `chore`    | 사소한 코드 변경                  |
| `fix`      | 버그 수정                        |
| `design`   | CSS 등 사용자 UI 디자인 변경      |
| `!HOTFIX`  | 긴급한 변경                      |
| `refactor` | 프로덕션 코드 리팩토링            |
| `docs`     | 문서 작성 및 수정                |
| `test`     | 테스트 추가 및 리팩토링           |
| `setting`  | 패키지 설치 등 개발 환경 설정     |
